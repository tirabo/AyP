1. Algunos ejemplos#

En esta primera parte veremos algunos ejemplos sencillos de recursión y recursión sobre listas.

Ejemplo (función recursiva). Sea la función recursiva

\[\begin{equation*} f(n) = \left\{ \begin{matrix} 1 \quad&\text{si $n=0$},\\ 2 \quad& \text{si $n=1$} \\ 2f(n-1) -3f(n-2) & \text{si $n>1$} \end{matrix} \right. \end{equation*}\]

  1. Implementar la función en Python según su definición.

  2. Definir una función recursiva en Python que no utilice recursión doble y permita recuperar \(f\).

Solución.

def f(n: int) -> int:
    """
    pre: n es entero >= 0
    post: devuelve f(n) definida más arriba
    """
    assert type(n) == int and n >= 0, 'el input debe ser un entero no negativo'
    if n == 0:
        return 1
    elif n == 1:
        return 2
    else:
        return  2*f(n-1) -3*f(n-2)

def f_s(n: int) -> tuple:
    """
    pre: n es entero >= 0
    post: devuelve (f(n), f(n+1)) con f definida más arriba
    """
    assert type(n) == int and n >= 0, 'el input debe ser un entero no negativo'
    if n == 0:
        return (1, 2)
    else:
        # f_s(n-1) == (f(n-1), f(n))
        # f_s(n) == (f(n), f(n+1) = 2*f(n) -3*f(n-1) )
        (fn_1, fn) = f_s(n-1)
        return (fn, 2*fn -3*fn_1)

Luego f_s() no tiene recursión doble y f(n) = f_s(n)[0].

Ejemplo (recursión sobre listas). Implemente recursivamente la función maximo() que toma una lista no vacía de enteros y devuelve su máximo.

Obviamente, no se puede usar la función max(), ni cualquier otra función que no sea elemental.

Solución.

def maximo(lista: list) -> int:
    """
    pre: lista es una lista de enteros
    post: devuelve el máximo de la lista
    """
    assert type(lista) == list and all(type(x) == int  for x in lista) and len(lista) > 0,'el input debe ser una lista no vacía de enteros.'
    if len(lista) == 1:
        return lista[0]
    else:
        max_cola = maximo(lista[1:])
        if lista[0] >= max_cola:
            return lista[0]
        else:
            return max_cola

2. Ejemplo de recursión: torres de Hanoi#

Las Torres de Hanói es un rompecabezas o juego matemático inventado en 1883 por el matemático francés Édouard Lucas.

Es un juego de mesa y lo describimos a continuación: contamos con tres postes verticales y \(n\) discos de distinto tamaño (\(n \in \mathbb N\)). En el primer poste se apilan los \(n\) discos de mayor a menor (el más grande abajo, el más chico arriba). Los otros dos postes quedan vacíos. El objetivo del juego consiste en pasar todos los discos desde el poste ocupado a uno de los otros postes vacíos. Para completar este objetivo, es necesario seguir tres simples reglas:

  1. Solo se puede mover un disco cada vez de un poste a otro.

  2. En cada poste los discos deben estar ordenados de mayor a menor desde la base.

  3. Solo se puede desplazar el disco que se encuentre arriba en cada poste.

Veamos una posible solución con 4 discos:

  1. Comezamos con la posición original.

  1. Pasamos el primer disco a la 2º pila.

  1. Pasamos el disco de arriba de la 1º pila a la 3º pila.

  1. Pasamos el disco de la 2º pila a la 3º pila.

  1. Pasamos el disco de arriba de la 1º pila a la 2º pila.

  1. Pasamos el disco de arriba de la 3º pila a la 1º pila.

  1. Pasamos el disco de de la 3º pila a la 2º pila.

  1. Pasamos el disco de arriba de la 1º pila a la 2º pila.

  1. Pasamos el disco de la 1º pila a la 3º pila.

  1. Pasamos el disco de arriba de la 2º pila a la 3º pila.

  1. Pasamos el disco de arriba de la 2º pila a la 1º pila.

  1. Pasamos el disco de arriba de la 3º pila a la 1º pila.

  1. Pasamos el disco de la 2º pila a la 3º pila.

  1. Pasamos el disco de arriba de la 1º pila a la 2º pila.

  1. Pasamos el disco de la 1º pila a la 3º pila.

  1. Pasamos el disco de la 2º pila a la 3º pila.

Estos 16 pasos (si contamos no hacer nada como el primer paso) que requiere el problema para 4 discos responden a un algoritmo recursivo que explicaremos a continuación.

Tenemos 3 pilas que llamaremos origen, auxiliar y destino. El objetivo es mover los \(n\) discos de origen a destino.

Caso base. Si \(n = 1\) solo se mueve el único disco de origen a destino.

Paso recursivo. En este caso suponemos que conocemos el algoritmo para mover \(n-1\) discos de una pila a otra y ahora, usando este algoritmo, debemos mover \(n\) discos.

  1. Mover los \(n-1\) discos de arriba de la pila origen a la pila auxiliar.

  2. Mover el disco que queda (el más grande) de la pila origen a la pila destino.

  3. Mover los \(n-1\) discos de la pila auxiliar a la pila destino.

Haciendo estos tres pasos el problema está resuelto.

Este algoritmo así presentado es claro que funciona (“demostración por inducción”). Tampoco es difícil probar por inducción que requiere \(2^{n}\) pasos, es decir que para \(n\) grande es inaplicable. Pero eso es inherente al problema: la complejidad de resolver este problema es del orden \(2^n\), se haga como se haga.

El algoritmo presentado arriba está explicado coloquialmente, ahora veamos como, abstrayendo un poco, lo podemos implementar en Python.

Primero, las tres pilas las pensamos como listas y los discos los podemos pensar como números enteros de \(1\) a \(n\).

Antes de hacer el algorimo escribamos la signatura y precondiciones y postcondiciones de la función. Debemos hacer una función que nos permita mover los \(n\) discos de arriba (puede haber más, pero siempre bien ordenados) de la pila origen a la pila destino (que puede no estar vacía, pero tiene discos más grandes que la de origen).

3. Quicksort#

Quicksort es un algoritmo de ordenamiento que fue desarrollado por Tony Hoare en 1959. Es uno de los algoritmos de ordenamiento más utilizados debido a su eficiencia y su simplicidad.

El algoritmo quicksort se implementa con recursión sobre listas. La idea básica detrás de quicksort es la siguiente: se elige un elemento de la lista, conocido como pivote, y se reorganizan los elementos de la lista de manera que todos los elementos menores que el pivote estén a su izquierda y todos los elementos mayores que el pivote estén a su derecha. De esta forma, se logra que el pivote esté en su ubicación definitiva, es decir, la ubicación que tendría cuando la lista esté completamente ordenada.

Luego, se repite el proceso en las sublistas generadas a cada lado del pivote, obteniéndose nuevos pivotes en su lugar definitivo. Si aplicamos repetidamente este proceso terminaremos con la lista ordenada.

El algoritmo se puede implementar de varias formas, pero una implementación típica de Quicksort funciona de la siguiente manera:

  1. Se elige un elemento de la lista como pivote.

  2. Se reorganizan los elementos de la lista de manera que todos los elementos menores que el pivote estén a su izquierda y todos los elementos mayores que el pivote estén a su derecha. Esto se conoce como pivoteo de la lista.

  3. Se llama recursivamente al algoritmo quicksort en las sublistas generadas a cada lado del pivote.

El pivoteo de la lista es la parte crítica del algoritmo. Una implementación típica utiliza una técnica conocida como “partición de Hoare” para pivotear la lista en lugar de crear nuevas sublistas. La partición de Hoare mueve dos índices desde los extremos izquierdo y derecho de la lista hacia el centro, intercambiando elementos mientras se acercan. Cuando los índices se cruzan, se ha completado el pivoteo de la lista.

Quicksort es un algoritmo muy eficiente en términos de tiempo de ejecución, con una complejidad de tiempo promedio de \(O(n \cdot \operatorname{log}(n))\) para listas de tamaño \(n\). Los algoritmos más intuitivos de ordenamiento (por selección y por inserción) tienen una complejidad \(O(n^2)\). Puede parecer menor la diferencia pero basta ejemplificar con una tabla la cantidad aproximada de pasos necesaria en cada caso:

Dejaremos la partición de Hoare para más adelante, ahora veremos una versión más simple, usando listas por comprensión y recursión sobre listas. Explicamos como es el procedimiento para ordenar una lista xs:

  1. Elegimos como pivote el primer elemento de la lista: pivote = xs[0].

  2. Pivoteamos las listas usando comprensión:

     xs_izq = [elem for elem in xs[1:] if elem <= pivote]
 xs_der = [elem for elem in xs[1:] if elem > pivote]

  3. Como estas listas son de longitud menor que la lista original, podemos utilizar inducción, es decir ordenamos por quicksort xs_izq y xs_der y luego concatenando obtenemos la lista ordenada xs_izq + [pivote] + xs_der.

Veamos la implementación.

def quicksort_1(xs):
    """
    pre: xs es una lista de números reales
    post: usando quicksort devuelve una lista ordenada con los elementos de xs

    """
    # hi = len(ls) - 1 if hi == -1 else hi
    # print(ls)
    if len(xs) <= 1:
        return xs
    else:
        pivote = xs[0]
        xs_izq = [elem for elem in xs[1:] if elem <= pivote]
        xs_der = [elem for elem in xs[1:] if elem > pivote]
        # Particionar la lista y obtener el índice pivote
        return quicksort_1(xs_izq) + [pivote] + quicksort_1(xs_der)

Esta implementación es correcta, pero no cumple con la idea original de “reordenar el arreglo” pues lo que hace es crear arreglos nuevos todo el tiempo. En problemas donde se manejan cantidades grandes de datos no es admisible duplicar o recrear arreglos, pues esos procedimientos comprometerían la memoria disponible.

Veamos a continuación una implementación de quicksort donde se trabaja con un solo arreglo y definida de manera similar a como fue implementada por Hoare.

Lo primero que debemos hacer es definir la función que pivotea el arreglo.

Definiremos

particion(ls, i, j)

de la siguiente manera:

  1. Tomamos como pivot p el primer elemento. La sublista o subarrglo que vamos a pivotear va del índice i al índice j.

  2. Un índice k va a indicar la posición del pivot. Al comienzo k = i. Siempre se va a cumplir que ls[h] < p si h < k. Es decir, todos los elementos entre el índice i y el índice k-1 son menores que p = ls[k].

  3. Un índice r va a indicar que ls[h] >= p para h > r. Al comienzo r = j. Es decir, todos los elementos entre el índice r+1 y el índice j son mayores o iguales que p = ls[k].

  4. Iteramos mientras k < r.

    • Si ls[k+1] < p permutamos p y ls[k+1] y hacemos k = k + 1.

    • Si ls[k+1] >= p permutamos ls[k+1] y ls[r] y hacemos r = r - 1

La froma en que estamos interando nos garantiza que las condiciones 2. y 3. se siguen cumpliendo. En cada paso iterativo o bien aumenta el k o bien disminuye el r, y esto nos garantiza que la iteración en algún momento finalizará (cuando k >= r).

La siguiente es una implementación de la función particion():

def particion(ls, i, j):
    """
    pre: ls es una lista de números reales, 0 <= i <= j < len(ls), todos distintos
    post: Sea p = ls[0] (pivote).  Permuta los elementos del arreglo ls[i:j+1]
          y devuelve un valor k tal que,
          a. i <= k <= j,
          b. ls[h] < p para todo i <= h < k,
          c. ls[k]= p, y
          d. ls[h] >= p para todo r < h <= j.
    """

    p = ls[i]
    k, r = i, j
    # Invariante
    # 1. ls[k] == p,
    # 2. ls[h] < p para todo i <= h < k <= j, y
    # 3. ls[h] >= p para todo i <= r < h <= j.
    while k < r:
        if  ls[k+1] < p:
            ls[k], ls[k+1] = ls[k+1], ls[k]
            k += 1
            # Sigue valiendo 1. y 2. (3. no fue tocado)
        else:
            ls[r], ls[k+1] = ls[k+1], ls[r]
            r -=1
            # Sigue valiendo 3. (1, y 2. no se modificaron)
    return k

Observar que terminado el cíclo vale

  1. ls[h] < p para todo i <= h < k,

  2. ls[k]= p,

  3. ls[h] >= p para todo r < h <= j, y

  4. k >= r

Luego combinadndo .2, 3. 4. obtenemos un nuevo 3.:

  1. ls[h] < p para todo i <= h < k,

  2. ls[k]= p,

  3. ls[h] >= p para todo k <= h <= j.

Es decir ls está “partido” por p.

Observa que la función de partición no es recursiva.

Ahora implementamos recursivamente quicksort:

def quicksort(ls, lo, hi):
    """
    pre: ls es una lista de números reales
    post: ordena la lista usando quicksort
    mod: ls
    """
    if lo < hi:
        # Particionar la lista y obtener el índice pivote
        r = particion(ls, lo, hi)
        # Ordenar las dos particiones
        quicksort(ls, lo, r - 1) # Lado izquierdo del pivote
        quicksort(ls, r + 1, hi) # Lado derecho del pivote

4. Criba de Eratóstenes con recursión#

La criba de Eratóstenes es un método antiguo para encontrar todos los números primos menores a un número dado. El proceso comienza con el número \(2\), que es el primer número primo. Luego, se van eliminando los múltiplos de cada número primo encontrado hasta que se llega al número dado.

Por ejemplo, si queremos encontrar los números primos menores a \(100\), comencemos con una lista con los números de \(2\) a \(99\) en orden. Comenzamos con el número \(2\), que es primo y eliminamos todos los múltiplos de \(2\). El próximo número de la lista, que es el \(3\), es primo y ahora eliminamos todos los múltiplos de él, este proceso se repite hasta haber recorrido toda la lista.

Comencemos a implementar la criba definiendo la función que elimnima los múltiplos:

def filtrar_multiplos(lst, n, contador):
    """
    pre:    lst : lista de numeros enteros positivos
            n : entero positivo
            contador : entero positivo
    post:   remueve de lst los multiplos de n, distintos de n y con indice
            mayor igual que contador
    """
    if len(lst) <= contador:
        pass
    else:
        if lst[contador] % n == 0 and lst[contador] != n:
            lst.remove(lst[contador])
        filtrar_multiplos(lst, n, contador+1)

Por lo tanto si hacemos

lista = list(range(1,m + 1))
filtar_multiplos(lista, n, 0)

Obtenemos la lista de enteros de 1 a m menos los multiplos de n mayores que n.

Por ejemplo,

lim = 100
lst = list(range(1,lim+1))
n, counter = 3, 0
filtrar_multiplos(lst, n, counter)
print(lst) # enteros de 1 a 100 menos los múltiplos de 3 mayores que 3

Podemos implementar la criba de Eratóstenes iterando sobre la función anterior.

def criba(m):
    lista = list(range(1,m + 1))
    if m == 1:
        return []
    else:
        for n in range(2, int(m**0.5)):
            filtrar_multiplos(lista,n,0)
        return lista

print(criba(50))

5. Ejemplo: el número combinatorio (2º parte)#

Recordemos.

Definición. Sean \(n\), \(m\) números naturales o 0 tal que \(m \le n\), definimos el número combinatorio \(n\), \(m\) denotado \(\displaystyle\binom{n}{m}\), por:

\[\binom{n}{m} = \frac{n!}{(n-m)!m! }.\]

Una forma de calcular el número combinatorio sería calcular \(n!\), \(m!\), \((n-m)!\) y después hacer las multiplicaciones y divisiones correspondientes.

Usando propiedades de los números combinatorios, podemos deducir que si \(n\ge 0\), entero:

  • \(\displaystyle\binom{n}{0} = 1\)

  • \(\displaystyle\binom{n}{n} = 1\)

  • \(\displaystyle\binom{n}{m}= \displaystyle\binom{n-1}{m-1} + \displaystyle\binom{n-1}{m}\), si \(0< m< n\).

A nosotros nos interesan estas ecuaciones porque podemos tomarlo como una definición recursiva del número combinatorio y así obtener una implementación recursiva del mismo:

def combinatorio(n, m: int) -> int:
    """
    pre: n >= 0 and 0 <= m <= n
    post: devuelve el número combinatorio de "n elementos tomados de a m"
    """
    assert type(n) == type(m) == int and n >= 0 and 0 <= m <= n, 'Error en los parámetros de la función "combinatorio"'
    if m == 0 or m == n:
        res = 1
    else:
        res = combinatorio(n - 1, m - 1) + combinatorio(n - 1, m)
    return res

Este es un buen ejemplo de recursión, pero no es para nada eficiente. Como en el caso de la sucesión de Fibonacci, la llamada recursiva a la función en dos valores (recursión doble) aumenta enormemente el tiempo de ejecución para casos no demasiado grandes.

print(combinatorio(15, 3))  # no hay problema
# combinatorio(40,12) # no lo puede hacer

Habíamos dejado como ejercicio definir combinatorio() con programación dinámica.

Resolvamos el ejericio.

En este caso habrá que usar una lista de listas, es decir una lista donde en cada coordenada hay una lista. Supongambos que cmb es una lista de listas, entonces cmb[i] es una lista y cmb[i][j] es la coordenada j de la lista cmb[i]. En estos caso suele hablarse de la coordenada i,j de cmb. Si la lista cmb tiene longitud \(n\) y cada lista cmb[i] tiene longitud \(m\), suele decirse que cmb es un arreglo \(n \times m\).

Para implementar combinatorio(n) con programación dinámica usaremos una lista de listas, un arreglo \((n + 1) \times (n+1) \), que llamaremos cmb y tal que cmb[i][j] sea el número combinatorio \(\binom{i}{j}\), cuando \(0 \le j \le i \le n\). Por ejemplo, cmb[5][2] == 10.

def combinatorio(n, m: int) -> int:
    """
    pre: n >= 0 and 0 <= m <= n
    post: devuelve el número combinatorio de "n elementos tomados de a m"
    """
    cmb = []
    for i in range(n+1):
        cmb.append([0]*(n+1))
    # cmb lista de listas con 0 en los coeficientes
    # cmb[i][j] = 0  con 0 <= i <= n, 0  <= j <= n
    # Luego  será cmb[i][j] = el combinatorio i,j para 0  <= j <= i <= n
    # print(cmb) # descomentar si se quiere ver la estructura de cmb
    for i in range(n+1):
        cmb[i][i] = 1 # los "caso base", los lados del triángulo de Pascal
        cmb[i][0] = 1
    for i in range(2, n+1):
        for j in range(1, i):
            cmb[i][j] = cmb[i-1][j-1] + cmb[i-1][j]
    # for i in range(n+1): # para "ver" el triángulo de Pascal
    #     print(cmb[i])
    return cmb[n][m]

def var(n, k):
    # pre: n, k enteros no negativos, n >= k.
    # post: devuelve n! / k!
    res = 1
    for i in range(k + 1, n + 1):
        res = res * i
    return res

def combinatorio_var(n, m):
    # pre: n >= 0, m >= 0, m <= n
    # post: devuelve el valor de n! / (m! * (n-m)!)
    return var(n, n - m) // var(m, 0)

Observación. En la definición con programación dinámica de la función combinatorio() creamos, al comienzo, un arreglo \((n+1) \times (n+1)\) con el siguiente código:

    cmb = []
    for i in range(n+1):
        cmb.append([0]*(n+1))

es decir creamos una lista de longitud \(n+1\) donde en cada coordenada se encuentra [0]*(n+1), una lista de \(n+1\) ceros. ¿Por qué no hicimos directamente

cmb = [[0]*(n+1)]*(n+1)

que tiene las misma coordenadas? Veamos un ejemplo:

n = 4
cmb_1 = []
for i in range(n+1):
    cmb_1.append([0]*(n+1))
print('Primera forma de definir:', cmb_1)

cmb_2 = [[0]*(n+1)]*(n+1)
print('Segunda forma de definir:', cmb_2)

Parecn iguales pero no lo son. La razón es que esta segunda forma de definir pone la misma lista en cada coordenada. Nosotros queremos que cada valor del arreglo pueda ser trabajado independientemente, sin embargo con la última definición si se hace la asignación cmb[3][2] = 5, tendremos que cmb[3][0] == 5, cmb[3][1] == 5, cmb[3][2] == 5, cmb[3][3] == 5, etc. Veamos esto en el ejemplo:

cmb_1[3][2] = 5
print('Primera forma de definir:', cmb_1)

cmb_2[3][2] = 5
print('Segunda forma de definir:', cmb_2)
